Actúa como un Desarrollador Frontend Senior y Arquitecto de Sistemas de Información Geográfica (SIG). Tu objetivo es construir una aplicación web de una sola página (SPA) llamada "Geocodificador Asistido". 

La aplicación permite a los analistas de datos cargar bases de datos tabulares con direcciones, normalizarlas, agruparlas, geocodificarlas asistidamente en un mapa con validación espacial cruzada, y exportarlas en formatos estrictos para integrarlas en ArcGIS Enterprise.

### 1. STACK TECNOLÓGICO Y ARQUITECTURA BASE
- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3. NO usar frameworks (React/Vue) ni bundlers (Webpack/Vite). Debe ser desplegable arrastrando la carpeta a Netlify.
- **Modularidad:** Separación estricta de responsabilidades (ej. `app.js` para estado, `io.js` para parseo/exportación, `map.js` para Leaflet, `ui.js` para la interfaz).
- **Librerías externas (vía CDN):** `Leaflet.js` (Mapas), `Turf.js` (Análisis espacial en el navegador), `SheetJS / xlsx` (Lectura/Escritura de Excel).
- **Seguridad:** Arquitectura "Local-First". Procesamiento 100% en memoria RAM del navegador. Nada se envía a bases de datos externas.

### 2. FLUJO DE USUARIO (WIZARD DE 4 PASOS)
La interfaz se divide en 4 pasos secuenciales.

#### PASO 1: Carga de Datos
- **Inputs:** 1. Excel de direcciones (Requerido). 2. Excel Maestro de Localidades (Opcional). 3. Capa de Áreas GeoJSON (Polígonos comunales para validación). 4. Capa de Recintos GeoJSON (Diccionario visual y de códigos).

#### PASO 2: Normalización y Mapeo
- UI para homologar columnas del Excel cargado a variables internas (ID/RUN, Dirección, Localidad, Comuna).
- **Clustering Automático:** Agrupar registros con direcciones idénticas para no geocodificar lo mismo dos veces.
- **Pestaña "Sugerencias":** Algoritmo (ej. distancia de Levenshtein) para detectar direcciones similares por errores de tipeo y sugerir fusiones.

#### PASO 3: Validación Espacial (El Core)
- Pantalla dividida: Lista de clusters (Izquierda) y Mapa Leaflet + Google Street View (Derecha).
- **Lógica de Operación (Orden estricto):**
  1. **Revisión y Gestión:** Revisar nombre del cluster. Herramientas para "Separar" (si hay numeraciones distintas) o "Fusionar" clusters.
  2. **Definir Dominio (Precisión):** El usuario clasifica el cluster como: E (Exacto), C (Calle), L (Localidad) o N (No Geo).
  3. **Herramientas Dinámicas:** Según el dominio, el sistema ofrece:
     - *Nominatim (Para E o C):* Búsqueda libre donde el usuario puede modificar el texto de consulta para sortear errores del motor.
     - *Localidades (Para L):* Lista desplegable del archivo Maestro cargado. Incluye botón para "+ Crear nueva localidad manual" (se ancla a una lat/lon en el mapa y se marca internamente como origen 'manual').
     - *Ingreso Manual:* Input para pegar coordenadas de Google Maps/SII.
  4. **Validación Turf.js:** Al confirmar, si hay un GeoJSON de Áreas cargado, evaluar Point-in-Polygon. Si el punto cae fuera de la comuna del registro original, lanzar alerta advirtiendo la inconsistencia.

#### PASO 4: Exportación de Resultados
Panel con 4 descargas asíncronas independientes:
1. **Capa para ArcGIS (Esri JSON):** [Botón Principal]. Exporta el esquema nativo de Esri (`esriGeometryPoint`, `esriFieldTypeInteger/Double`). Convierte nulos de texto a `null` real. Clave para evitar el "Error 160117" en ArcGIS Pro.
2. **Maestro de Localidades:** Exporta un Excel que consolida las localidades oficiales del Paso 1 MÁS las nuevas localidades manuales creadas en el Paso 3, conservando todas las columnas del Excel original.
3. **Excel para Append:** Tabla simplificada (ID, Dominio numérico, Lat, Lon).
4. **Respaldo Completo (Guardar Partida):** Clona el Excel de direcciones original (conservando TODAS sus columnas) e inyecta las coordenadas y dominios nuevos.

### 3. REGLAS DE NEGOCIO Y RESTRICCIONES CRÍTICAS
1. **La Regla "NO GEO":** Si un cluster se marca como dominio "N" (No Geo), el exportador ESTÁ OBLIGADO a ignorar la vista del mapa y debe restaurar/exportar las coordenadas exactas originales que traía el Excel en el Paso 1.
2. **La "Supermente":** Sistema para importar/exportar un JSON ligero con las coordenadas confirmadas como "Exactas". Al importar una Supermente, el sistema debe cruzarla con el Excel cargado y autocompletar instantáneamente los clusters coincidentes para evitar trabajo duplicado entre analistas.
3. **Guardián de Sesión:** Evento `beforeunload` que advierte al usuario si intenta cerrar la pestaña teniendo datos sin exportar.
